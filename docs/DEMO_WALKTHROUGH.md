# One-Page Demo Walkthrough (Schools and NGOs)

## Goal
Show how EduShelf runs a school library smoothly with or without internet, while supporting inclusive, remote-friendly operations.

## Time required
10–15 minutes

## Setup
- Server running on the demo laptop.
- Desktop app installed on a staff laptop.
- Optional: Use "Seed Demo Data" on the Dashboard.

## Demo flow

1. Login
   - Role-based access (Admin or Operator).
   - Emphasize security and accountability.

2. Dashboard overview
   - Daily snapshot (issued, returns, overdue, pending sync).
   - AI summary (optional) using local Ollama.

3. Quick Add (speed and simplicity)
   - Add a new book with minimal fields.
   - Add a new member in under 30 seconds.

4. Issue and return
   - Issue a book to a student.
   - Auto due date and fine preview.
   - Return the book and show fine calculation.

5. Reports and analytics
   - Export Books PDF with school branding.
   - Export Loans Excel in the approved format.
   - Show Top Books usage chart.

6. Offline-first story
   - Explain that every action is stored locally.
   - Sync happens when internet returns.
   - Sync queue shows pending and conflicts clearly.

## Key messages (for principals and NGO partners)
- Works without reliable internet.
- Supports librarians with mobility challenges.
- Faster operations and better accountability.
- Easy reporting for school leadership and CSR reporting.

## Suggested closing
“If a librarian cannot reach the library, EduShelf brings the library to them.”
